# Technical SEO Audit & Fixes — EDUNEX Services

**Domain:** https://www.edunexservices.in  
**Audit Date:** March 2026  
**Status:** Implemented

---

## 1. Critical SEO Errors (Fixed)

| Issue | Priority | Root Cause | Fix Applied |
|-------|----------|------------|-------------|
| Canonical/host mismatch | Critical | Metadata and sitemap used `https://edunexservices.in` while live traffic is on `https://www.edunexservices.in` | Switched all canonicals, metadataBase, sitemap, and robots to `https://www.edunexservices.in` via `SITE_URL` |
| Redirect inconsistency | Critical | Next.js redirected www → apex while platform redirects apex → www | Reversed redirect: apex → www (301) |
| Invalid URLs returning 200 | Critical | Blog and case-study invalid slugs rendered custom "Not Found" UI instead of calling `notFound()` | Use `notFound()` for blog; case-study converted to server component with `notFound()` |
| Broken internal links | Critical | Header/footer service links used wrong slugs (`web-app-development`, `crm-erp-solutions`, `digital-marketing-services`) | Updated to real slugs: `custom-software-bihar`, `erp-billing-software`, `local-seo-marketing` |
| Resources page non-crawlable blog links | Critical | Blog cards and "View All Posts" were non-link elements | Wrapped in `<Link>` to `/blog` and `/blog/[slug]` |
| Sitemap unstable lastmod | Medium | `new Date()` on every request made lastmod always "now" | Use stable `staticLastMod`; blog posts use parsed date |
| Service title HTML in metadata | Medium | `outcomeHeadline` contained `<span>` tags rendered in `<title>` | Strip HTML with `stripHtml()` before metadata |
| Blog schema wrong date | Medium | `datePublished` hardcoded as "2024-01-01" | Use post.date parsed to ISO (YYYY-MM-DD) |
| Homepage case-study cards | Medium | Linked only to `/case-studies` archive | Link to `/case-studies/[slug]` for each card |
| Navigation schema missing Blog/Resources | Low | ItemList omitted Blog and Resources | Added Blog and Resources to navigation schema |

## 2. Step-by-Step Fix Plan (Completed)

1. **Canonical host** — Centralized `SITE_URL` in `src/config/company.ts`; updated layout, all layouts, dynamic metadata, schema components, sitemap, and robots.
2. **Redirect** — `next.config.mjs`: apex host → `https://www.edunexservices.in` (301).
3. **404 handling** — Blog: `notFound()` when post missing. Case-studies: server `page.tsx` with `notFound()`, client `CaseStudyDetailClient` for UI.
4. **Sitemap/robots** — `SITE_URL`; stable `lastModified`; blog dates parsed for lastmod.
5. **Internal links** — Header, Footer: correct service slugs; Resources: Link for View All and each blog card; Homepage: case-study links to detail; case-study detail: link to Services.
6. **Metadata** — Service title: `stripHtml(outcomeHeadline)`; Blog schema: ISO date.
7. **Loading** — Root loading UI converted to server component (no motion client dependency).

## 4. Correct Sitemap Structure

- **Base URL:** `https://www.edunexservices.in`
- **Static routes:** /, /about, /blog, /case-studies, /compliance, /contact, /cookies, /privacy, /resources, /services, /solutions, /terms

## 8. Google Search Console Categories — Root Causes & Fixes

| Category | Root Cause | Fix |
|----------|------------|-----|
| **Duplicate without user-selected canonical** | Canonicals pointed to apex while pages served on www; conflicting signals | All canonicals now point to www |
| **Page with redirect** | Sitemap listed apex URLs that redirected to www | Sitemap now lists www URLs only |
| **Alternative page with proper canonical** | www pages canonically pointed to apex; Google treated them as alternates | Canonicals now self-referencing on www |
| **Discovered – currently not indexed** | Soft 404s (200 for invalid URLs), weak internal links, possible crawl depth | Proper 404 for invalid slugs; stronger internal links and crawl paths |
